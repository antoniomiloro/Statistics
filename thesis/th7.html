Th7: "Online" Algorithms (Data Streams): Ideas and code

A stream of data refers to a continuous and potentially infinite flow of information that is generated and processed over time. Instead of being presented as a static set, a data stream continuously evolves. Here are some key characteristics of data streams:
Examples of data streams include:

Sensor Data: Data from IoT devices, such as temperature sensors, can be streamed in real-time.
Social Media Feeds: photos and posts on platforms like Instagram and Facebook form continuous streams of data.
Financial Transactions: Stock market data, currency exchange rates forming data streams.
Network Traffic: Data streams in the context of network monitoring include the continuous flow of packets and information across a network.

Managing and analyzing data streams require specialized techniques and algorithms that can handle the challenges posed by the dynamic and continuous nature of the data. Online algorithms are one example of techniques designed for processing data streams efficiently.

In computer science, an Online Algorithm is one that can process its input piece-by-piece in a serial fashion, i.e., in the order that the input is fed to the algorithm, without having the entire input available from the start.
In contrast, an offline algorithm is given the whole problem data from the beginning and is required to output an answer which solves the problem at hand.
In operations research, the area in which online algorithms are developed is called online optimization.
As an example, consider the sorting algorithms selection sort and insertion sort: selection sort repeatedly selects the minimum element from the unsorted remainder and places it at the front, which requires access to the entire input; it is thus an offline algorithm.
On the other hand, insertion sort considers one input element per iteration and produces a partial solution without considering future elements. Thus insertion sort is an online algorithm.
Note that the final result of an insertion sort is optimum, i.e., a correctly sorted list. For many problems, online algorithms cannot match the performance of offline algorithms.
If the ratio between the performance of an online algorithm and an optimal offline algorithm is bounded, the online algorithm is called competitive.
Not every offline algorithm has an efficient online counterpart.

Characteristics of Data Streams:

Unbounded Data: Data streams are continuous and unending, making it impossible to store all data in memory.
Limited Memory: Algorithms must operate within constrained memory limits.
Single Pass: Typically, data is processed only once as it arrives and cannot be revisited.

For example we could compute the mean (average) of a data stream without storing all individual values.

class OnlineMean:
def __init__(self):
self.count = 0 # Number of elements processed
self.mean = 0.0 # Current mean

def update_mean(self, new_value):
self.count += 1
delta = new_value - self.mean
self.mean += delta / self.count

def get_mean(self):
return self.mean

the previous python code works as follows:

Initialization: Initialize count as 0 and mean as 0.0.
Update Mean: As each new value arrives, update the mean using the formula:
new_mean = old_mean + (new_value - old_mean) / count
Get Mean: Return the current mean when required.

Some of the fields where online algorithms are used are:

Streaming Analytics: Real-time monitoring of data (e.g., network traffic, sensor data).
Financial Data Processing: Analyzing stock market data streams for trends or anomalies.
Machine Learning: Online learning algorithms for continuously evolving models using streaming data.

Online algorithms for data streams offer efficient solutions to process continuous data while maintaining limited memory usage. These algorithms provide approximate solutions for various computational problems, making them crucial in handling massive and continuous data flows.
By employing techniques like approximation, sampling, and summarization, online algorithms enable the extraction of valuable insights from data streams while coping with the challenges posed by their infinite and voluminous nature.
